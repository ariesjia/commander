## 1. 产品概述

### 1.1 产品名称

**MotiMech**

### 1.2 一句话描述

一个面向家庭场景的学生游戏化激励 Web 应用——家长配置学习任务和奖励，学生通过完成任务赚取积分，驱动一台专属机甲从散落零件逐步组装为完整的变形战士，积分也可以兑换家长设定的真实奖励。

### 1.3 产品背景

家庭教育中，家长常面临以下困境：

- **口头约定难执行**："写完作业就让你玩"缺乏可追踪的记录
- **奖惩机制不透明**：孩子不清楚"做到什么程度能获得什么"
- **激励手段单一**：实物奖励成本高，口头表扬效果递减
- **缺乏可视化成果**：学习努力的累积过程不可见

本产品以男生喜爱的"机甲/变形金刚"主题为载体，将家庭任务激励体系游戏化，让积分的积累变成一台机甲从零到完整的养成过程。

### 1.4 产品目标

| 目标维度       | 具体指标                           |
| -------------- | ---------------------------------- |
| **任务完成率** | 家长配置的每日任务完成率 > 80%     |
| **持续使用**   | 家庭持续使用 30 天以上的比例 > 50% |
| **亲子互动**   | 家长每周至少登录确认 5 次          |
| **学生满意度** | 学生主观满意度评分 > 4/5           |

### 1.5 产品边界

| 包含                            | 不包含                       |
| ------------------------------- | ---------------------------- |
| 家长注册、登录、PIN 码设置      | 教师/学校管理端              |
| 同应用内家长/学生模式切换       | 学生独立设备访问（后续考虑） |
| 家长配置任务与奖励              | 自动化任务完成检测           |
| 学生查看任务、机甲养成          | 学生之间的社交互动           |
| 学生兑换奖励                    | 支付/充值功能                |
| 家长确认任务完成与奖励兑换      | 多孩家庭（后续版本）         |
| Web 响应式（iPad / 手机）       | 原生 App                     |
| 两套视觉风格（简洁 + 科技卡通） | —                            |

## 2. 目标用户与使用场景

### 2.1 角色定义

本系统只有 **两个角色**，共用同一个应用，通过**模式切换**区分：

| 角色     | 说明                                                 | 访问方式                                         |
| -------- | ---------------------------------------------------- | ------------------------------------------------ |
| **家长** | 注册账号，配置任务和奖励，确认任务完成，确认奖励兑换 | 邮箱注册登录 → 默认进入家长模式                  |
| **学生** | 查看任务，查看机甲状态，兑换奖励                     | 家长在应用内点击"切换到学生模式"，把设备交给孩子 |

**切换机制：**

- 家长登录后处于**家长模式**，可一键切换到**学生模式**
- 学生模式下，要切回家长模式需输入 **4 位 PIN 码**（家长注册时设置）
- 学生无法绕过 PIN 码进入家长模式，保证家长对任务/奖励的控制权

### 2.2 用户画像

**家长：张妈妈，38 岁**

- 希望孩子养成良好学习习惯，但缺乏有效工具
- 不想总是用"不许玩手机"来威胁
- 愿意花 5 分钟/天来确认孩子的任务完成情况
- 希望承诺的奖励有据可查，不会忘记

**学生：小明，11 岁，五年级男生**

- 喜欢变形金刚、乐高机械组、奥特曼
- 对"收集""升级""变形"天然有兴趣
- 希望自己的努力被"看见"
- 有明确的奖励目标时，做事更有动力

### 2.3 核心使用场景

| 场景           | 角色 | 描述                                       | 频率              |
| -------------- | ---- | ------------------------------------------ | ----------------- |
| 配置任务       | 家长 | 创建每日/每周任务，设定积分奖励            | 初始 + 每周调整   |
| 配置奖励       | 家长 | 创建可兑换的奖励项目及所需积分             | 初始 + 不定期调整 |
| 确认任务       | 家长 | 孩子完成任务后，家长点击确认               | 每天 2~5 次       |
| 切换到学生模式 | 家长 | 点击"切换到学生模式"，把设备递给孩子       | 每天 1~2 次       |
| 查看任务       | 学生 | 在学生模式下看今日任务和完成状态           | 每天 1~3 次       |
| 查看机甲       | 学生 | 看机甲的组装/进化进度                      | 每天 1~2 次       |
| 兑换奖励       | 学生 | 用积分兑换家长设定的奖励                   | 每周 0~2 次       |
| 确认兑换       | 家长 | 审核并确认学生的兑换请求（需切回家长模式） | 每周 0~2 次       |

## 3. 产品核心理念

### 3.1 设计原则

1. **家长主导，系统辅助** — 所有任务、奖励、确认都由家长掌控，系统只是让这个过程更有趣
2. **学习是充能，不是负担** — 每次任务完成都转化为可见的机甲成长
3. **积分可见，去向透明** — 积分怎么来的、花在哪了，家长和学生都一目了然
4. **操作极简** — 家长端操作控制在"3 次点击内"完成，学生端以"看"为主
5. **仪式感至上** — 机甲组装、进化的关键时刻要有动画和音效反馈

### 3.2 游戏化机制映射

| 游戏化机制              | 在本产品中的体现                                 |
| ----------------------- | ------------------------------------------------ |
| 进度反馈（Progress）    | 机甲组装进度条、积分累积                         |
| 养成系统（Nurture）     | 主力机甲从零件到完整体的组装过程                 |
| 进化升级（Evolution）   | 机甲 5 级进化 + 变形解锁                         |
| 目标驱动（Goal）        | 奖励商城中的目标（"再攒 50 分就能换游乐园门票"） |
| 连续激励（Streak）      | 连续完成任务天数                                 |
| 自定义（Customization） | 机甲涂装、命名                                   |

## 4. 系统架构总览

### 4.1 单应用双模式结构

本产品是一个**单应用**，通过**模式切换**实现家长端和学生端的分离。两种模式拥有完全不同的 UI 风格和功能集。

```
┌───────────────────────────────────────────────────────────────┐
│                     MotiMech（单应用）                          │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │                    家长模式                               │  │
│  │              风格：简洁 / 清晰 / 明了                      │  │
│  │                                                         │  │
│  │  • Dashboard 总览     • 任务管理 & 确认                   │  │
│  │  • 奖励管理 & 审核    • 积分记录                          │  │
│  │  • 设置（PIN 码 / 昵称）                                  │  │
│  │                                                         │  │
│  │              [ 切换到学生模式 👦 ]  ← 一键切换              │  │
│  └─────────────────────┬───────────────────────────────────┘  │
│                        │                                      │
│                   无需密码 ↓ ↑ 需输入 PIN 码                    │
│                        │                                      │
│  ┌─────────────────────┴───────────────────────────────────┐  │
│  │                    学生模式                               │  │
│  │              风格：科技 / 卡通 / 炫酷                      │  │
│  │                                                         │  │
│  │  • 机甲养成（核心视觉）  • 任务列表                        │  │
│  │  • 奖励商城 & 兑换      • 积分记录                        │  │
│  │                                                         │  │
│  │         [ 🔒 切回家长模式 ] ← 需输入 4 位 PIN              │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │                  Vercel 全栈服务                          │  │
│  │            Next.js App Router + API Routes               │  │
│  │                 Vercel Postgres                          │  │
│  └─────────────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────────────┘
```

### 4.2 模式切换流程

```
家长注册（邮箱 + 密码 + 4 位 PIN 码）
  → 登录（邮箱 + 密码）
  → 进入家长模式（默认）
  → 点击"切换到学生模式" → 立即切换（无需密码）
  → 学生使用中...
  → 点击"切回家长模式" → 弹出 PIN 输入框 → 输入正确 → 返回家长模式
                                          → 输入错误 → 提示重试（连续 5 次错误锁定 1 分钟）
```

**典型场景**：家长在手机/iPad 上登录 → 配置好任务 → 切到学生模式 → 把设备递给孩子 → 孩子用完后还给家长 → 家长输入 PIN 切回。

### 4.3 模式切换设计要点

| 要点         | 说明                                                                                                |
| ------------ | --------------------------------------------------------------------------------------------------- |
| 切换入口位置 | 家长模式：顶部导航栏右侧显著按钮；学生模式：底部导航栏角落的小锁图标                                |
| 切换动画     | 从家长 → 学生：画面从白色淡出，科技蓝色背景淡入 + 机甲启动动画；从学生 → 家长：画面回到简洁白色界面 |
| PIN 码 UI    | 4 个圆形输入框，数字键盘，输入完成自动验证                                                          |
| 防误触       | 学生模式下的"切回"按钮做小、做低调，不引起孩子注意                                                  |
| 记住登录     | 浏览器保持登录态（7 天），家长不用每次都输邮箱密码                                                  |
| 安全性       | PIN 码仅防孩子，不需要高安全级别；5 次错误锁定 1 分钟即可                                           |

## 5. 家长端功能

### 5.1 注册与登录

**注册流程（3 步）：**

| 步骤 | 内容             | 说明                             |
| ---- | ---------------- | -------------------------------- |
| 1    | 邮箱 + 密码      | 账号凭据，用于登录               |
| 2    | 设置 4 位 PIN 码 | 用于学生模式切回家长模式的验证   |
| 3    | 输入孩子昵称     | 学生模式下显示的称呼（如"小明"） |

**登录方式：** 邮箱 + 密码（保持 7 天登录态）

**密码重置：** 通过邮箱重置

### 5.2 学生管理

| 功能点      | 说明                             |
| ----------- | -------------------------------- |
| 修改昵称    | 修改学生在学生模式下的显示名称   |
| 修改 PIN 码 | 修改切回家长模式时的 4 位 PIN 码 |
| 查看状态    | 查看学生当前积分、机甲状态概览   |

> MVP 阶段：一个家长账号对应**一个学生**。后续版本支持多个学生（多孩家庭）。

### 5.3 任务管理

#### 创建任务

家长可创建以下类型的任务：

| 字段     | 说明                 | 必填           |
| -------- | -------------------- | -------------- |
| 任务名称 | 如"完成今日数学作业" | 是             |
| 任务描述 | 补充说明（可选）     | 否             |
| 任务类型 | 每日任务 / 每周任务  | 是             |
| 积分奖励 | 完成后获得的积分数   | 是             |
| 是否启用 | 可暂时停用某个任务   | 是（默认启用） |

**任务类型规则：**

| 类型     | 重置周期          | 说明                             |
| -------- | ----------------- | -------------------------------- |
| 每日任务 | 每天 00:00 重置   | 每天可完成一次，过期未完成则跳过 |
| 每周任务 | 每周一 00:00 重置 | 每周可完成一次                   |

#### 确认任务完成

- 家长在任务列表中看到所有当前有效的任务
- 每个任务旁有"确认完成"按钮
- 点击确认后：
  - 任务状态变为"已完成"
  - 积分自动发放到学生账户
  - 学生端可立即看到积分变化和机甲反馈动画
- 已确认的任务不可撤销（防止纠纷）

#### 任务管理操作

| 操作      | 说明                                           |
| --------- | ---------------------------------------------- |
| 创建      | 新增任务                                       |
| 编辑      | 修改任务名称、描述、积分（不影响已完成的记录） |
| 停用/启用 | 暂时隐藏任务，不删除历史数据                   |
| 删除      | 永久删除任务（需二次确认）                     |

### 5.4 奖励管理

#### 创建奖励

家长可设定学生可兑换的奖励：

| 字段     | 说明                 | 必填           |
| -------- | -------------------- | -------------- |
| 奖励名称 | 如"去游乐园玩一天"   | 是             |
| 奖励描述 | 补充说明（可选）     | 否             |
| 所需积分 | 兑换该奖励消耗的积分 | 是             |
| 是否启用 | 可暂时下架某个奖励   | 是（默认启用） |

#### 兑换审核

- 学生发起兑换请求后，家长端收到待审核通知
- 家长可以：
  - **确认兑换**：扣除积分，奖励状态变为"已兑换"
  - **拒绝兑换**：积分不扣除，附带拒绝原因（可选）
- 待审核期间，对应积分处于"冻结"状态（不可用于其他兑换）

#### 奖励管理操作

| 操作      | 说明                                       |
| --------- | ------------------------------------------ |
| 创建      | 新增奖励                                   |
| 编辑      | 修改名称、描述、积分（不影响已兑换的记录） |
| 停用/启用 | 暂时下架奖励                               |
| 删除      | 永久删除奖励（需二次确认）                 |

### 5.5 积分记录

家长可查看完整的积分流水：

| 记录字段  | 说明                               |
| --------- | ---------------------------------- |
| 时间      | 积分变动时间                       |
| 类型      | 获得（任务完成）/ 消耗（奖励兑换） |
| 来源/去向 | 具体任务名称或奖励名称             |
| 变动数量  | +N / -N                            |
| 余额      | 变动后的积分余额                   |

### 5.6 数据总览（Dashboard）

家长首页显示关键数据：

- 学生当前积分余额
- 本周任务完成情况（已完成 / 总数）
- 待确认事项数量（待确认的任务 + 待审核的兑换）
- 机甲当前状态缩略图
- 连续活跃天数（Streak）
- **"切换到学生模式"按钮**（显著位置，带机甲小图标）

### 5.7 模式切换按钮

- 在家长模式的每个页面顶部导航栏，右侧始终显示 **"切换到学生模式"** 按钮
- 按钮样式：带科技感图标（小机甲头像），与家长端简洁风格做对比，暗示"点了就进入另一个世界"
- 点击后无需任何确认，直接切换到学生模式

## 6. 学生端功能

### 6.1 访问方式

- 家长在应用内点击"切换到学生模式"后，进入学生模式
- **学生不需要任何账号、密码或链接**，家长替他切好即可
- 学生模式下无法访问任何家长功能（任务编辑、奖励编辑、确认操作等）
- 切回家长模式需要输入 4 位 PIN 码

### 6.2 学生模式入口动画

从家长模式切换到学生模式时，播放一个简短的过渡动画：

```
家长界面白色淡出 → 深蓝色科技背景淡入 → 机甲从远处飞来
→ "MotiMech · {昵称}，欢迎回来！" → 进入学生首页
```

首次进入学生模式时，播放完整的引导动画（约 5 秒）；后续切换播放简短版（约 1.5 秒）。

### 6.3 首页 — 机甲展示

学生模式首页的核心是**机甲养成画面**：

- 占据页面主要区域，展示当前机甲的组装/进化状态
- 底部显示积分余额和组装进度
- 有新的积分到账时，播放积分飞入 + 机甲反馈动画
- 长按机甲可触发待机动作 / 变形动画（解锁后）
- 背景为深蓝色科技风 + 星空/电路板纹理

### 6.4 任务列表

学生可以查看家长配置的任务：

| 信息     | 说明                     |
| -------- | ------------------------ |
| 任务名称 | 显示任务内容             |
| 积分奖励 | 完成后可获得的积分       |
| 任务状态 | 待完成 / 已完成 / 已过期 |
| 任务类型 | 每日 / 每周 标签         |

**注意：学生端不能自行标记任务为完成，必须由家长确认。** 学生看到的状态流转：

```
待完成 → （家长确认后）→ 已完成 ✓（积分到账，播放动画）
待完成 → （到期未确认）→ 已过期（灰色显示）
```

### 6.5 奖励商城

学生可以浏览和兑换奖励：

| 信息     | 说明                     |
| -------- | ------------------------ |
| 奖励名称 | 显示奖励内容             |
| 所需积分 | 兑换需要的积分           |
| 可否兑换 | 积分充足且无冻结时可兑换 |
| 兑换按钮 | 点击后发起兑换请求       |

**兑换流程：**

```
学生点击"兑换" → 二次确认弹窗（"确定要花 XX 积分兑换 YY 吗？"）
  → 确认 → 积分冻结，状态变为"等待家长确认"
  → 家长确认 → 积分正式扣除，奖励状态变为"已兑换 ✓"
  → 家长拒绝 → 积分解冻，状态恢复，显示拒绝原因
```

### 6.6 积分记录

学生可查看自己的积分变动历史（只看，格式与家长端一致）。

### 6.7 我的兑换

学生可查看历史兑换记录及状态：

| 状态   | 说明                 |
| ------ | -------------------- |
| 待确认 | 已提交，等待家长审核 |
| 已兑换 | 家长已确认           |
| 已拒绝 | 家长拒绝，积分已退回 |

### 6.8 切回家长模式

- 在学生模式的底部导航栏最右侧，有一个**小锁图标**（低调设计，不吸引孩子注意）
- 点击后弹出 PIN 码输入界面（4 个圆形输入框 + 数字键盘）
- PIN 正确 → 过渡动画 → 回到家长模式
- PIN 错误 → 提示"密码错误"，连续 5 次错误锁定 1 分钟

## 7. 模块：机甲养成系统

### 7.1 概述

机甲养成是学生端的核心视觉体验。学生的积分累积会自动映射到机甲的组装和进化进度上。**积分的主要用途是兑换奖励，机甲成长是积分累积的附带视觉反馈——不消耗积分。**

> 设计理念：机甲进度反映的是"你累计赚了多少"，而非"你花了多少"。这样学生不必在"养机甲"和"换奖励"之间做取舍。

### 7.2 机甲组装阶段

机甲的组装进度绑定**累计获得积分总量**（而非当前余额）：

| 阶段 | 部位        | 累计积分门槛 | 视觉效果                           |
| ---- | ----------- | ------------ | ---------------------------------- |
| 0    | 未启动      | 0            | 散落零件 + 虚影轮廓                |
| 1    | 底盘 / 脚部 | 20           | 脚部结构成型                       |
| 2    | 腿部        | 50           | 腿部完成，机甲"站起来"             |
| 3    | 躯干        | 100          | 核心区域组装，能量核心亮起         |
| 4    | 左臂        | 150          | 左臂安装                           |
| 5    | 右臂 + 武器 | 200          | 右臂安装，武器上线                 |
| 6    | 头部        | 270          | 头部安装，**双眼亮起**（高光时刻） |
| 7    | 涂装完成    | 350          | 上色完成，完整体亮相！             |

### 7.3 机甲进化体系

完成组装后，继续累积积分触发进化：

```
Lv.1 侦察型（组装完成，累计 350 分）
  ↓  累计 700 分
Lv.2 战斗型（体型增大，武器升级）
  ↓  累计 1500 分
Lv.3 重装型（厚重装甲，视觉更硬朗）
  ↓  累计 3000 分
Lv.4 飞行型（解锁飞行部件，可变形为战斗机）
  ↓  累计 6000 分
Lv.5 终极型（最终形态，华丽特效，专属变形动画）
```

### 7.4 变形系统

- Lv.4+ 的机甲解锁"变形"能力
- 学生在机甲展示页长按 → 触发变形动画（机甲 ↔ 载具）
- 纯视觉体验，不消耗积分
- 变形动画 2~3 秒，含零件重组过渡效果

### 7.5 自定义系统

| 自定义项 | 解锁条件             | 说明                         |
| -------- | -------------------- | ---------------------------- |
| 机甲命名 | 阶段 6（头部安装后） | 给机甲取专属名字             |
| 涂装颜色 | 阶段 7（组装完成后） | 从预设配色方案中选择         |
| 武器样式 | Lv.2+                | 替换武器外观                 |
| 特效光环 | Lv.3+                | 能量光环颜色                 |
| 待机动作 | Lv.4+                | 闲置时的动作（挥拳、敬礼等） |

## 8. 模块：积分系统

### 8.1 概述

系统采用**单一货币：积分**。积分是连接任务（赚取）和奖励（消耗）的核心。

### 8.2 积分获取

积分**只能通过家长确认任务完成来获得**，没有其他获取途径。

| 触发条件                   | 说明                               |
| -------------------------- | ---------------------------------- |
| 家长点击"确认完成"某个任务 | 该任务的积分奖励自动发放到学生账户 |

### 8.3 积分消耗

| 消耗场景 | 说明                             |
| -------- | -------------------------------- |
| 兑换奖励 | 学生发起兑换请求，家长确认后扣除 |

### 8.4 积分状态

| 状态     | 说明                                         |
| -------- | -------------------------------------------- |
| 可用积分 | 可以用于兑换奖励                             |
| 冻结积分 | 已提交兑换请求但家长尚未确认，暂时不可用     |
| 累计积分 | 历史总获得积分（只增不减，用于机甲进度计算） |

**关键设计：机甲进度绑定"累计积分"，不受兑换消耗影响。** 这意味着学生兑换奖励不会导致机甲"退化"。

### 8.5 积分流水记录

每笔积分变动产生一条记录：

```
+10  完成任务"数学作业"            2026-03-11 18:30
+5   完成任务"阅读30分钟"          2026-03-11 19:00
-50  兑换奖励"看一集动画片"         2026-03-12 20:00
+15  完成任务"整理书桌"            2026-03-13 17:30
```

### 10.1 概述

奖励商城是学生"花积分"的地方。奖励内容完全由家长自定义，可以是虚拟特权，也可以是真实物品/活动。

### 10.2 奖励配置示例

## 11. 数值体系设计

### 11.1 积分获取预估

假设家长配置 3 个每日任务（均值 8 分/个）+ 2 个每周任务（均值 20 分/个）：

```
每日可获得: 3 × 8 = 24 积分
每周额外:  2 × 20 = 40 积分
每周总计:  24 × 7 + 40 = 208 积分（全部完成）
实际预估:  约 150 积分/周（按 70% 完成率）
```

### 11.2 养成节奏对标

| 里程碑                 | 累计积分门槛 | 预计耗时（70%完成率） |
| ---------------------- | ------------ | --------------------- |
| 机甲脚部完成（阶段 1） | 20           | ~1 天                 |
| 机甲站起来（阶段 2）   | 50           | ~2~3 天               |
| 机甲核心亮起（阶段 3） | 100          | ~5 天                 |
| 机甲完整体（阶段 7）   | 350          | ~2.5 周               |
| 首次进化 Lv.2          | 700          | ~5 周                 |
| 飞行型 Lv.4            | 3000         | ~5 个月               |
| 终极型 Lv.5            | 6000         | ~10 个月              |

### 11.3 设计目标

- **首个视觉反馈（阶段 1）** 在 **第 1 天** 就能看到
- **机甲完整体** 控制在 **2~3 周**，让学生尽快体验完整形态
- **首次进化** 对齐 **1~1.5 个月** 的学习节奏
- **终极形态** 需要 **一学年** 左右，作为长期目标

### 11.4 奖励定价建议

家长配置奖励时，系统可提供参考建议：

| 奖励级别             | 建议积分范围 | 获取耗时参考 |
| -------------------- | ------------ | ------------ |
| 小奖励（一集动画）   | 20~50        | 1~2 天       |
| 中奖励（游戏时间）   | 50~150       | 2~7 天       |
| 大奖励（玩具/出行）  | 150~500      | 1~4 周       |
| 超级奖励（特别活动） | 500+         | 1 个月+      |

### 11.5 Streak 机制

连续有任务被确认完成的天数：

| 连续天数 | 视觉效果                   |
| -------- | -------------------------- |
| 3 天     | 机甲脚下出现小火焰         |
| 7 天     | 火焰变大 + "坚持 7 天"徽章 |
| 14 天    | 机甲背后出现能量翅膀       |
| 30 天    | 金色光环 + "钢铁意志"称号  |

> Streak 只产生视觉效果，不影响积分数量（保持简单）。

## 12. 视觉与动效规范

### 12.1 美术风格

家长模式和学生模式采用**完全不同的视觉语言**，切换时产生强烈的对比感。

#### 家长模式 — 简洁 / 清晰 / 明了

| 要素       | 规范                                           |
| ---------- | ---------------------------------------------- |
| 整体风格   | 极简工具风，类似 Notion / Linear               |
| 背景色     | 白色 / 浅灰（`#FAFAFA`）                       |
| 主色调     | 蓝灰色（`#4A5568`），强调色用蓝色（`#3B82F6`） |
| 字体       | 系统默认无衬线字体，层级清晰                   |
| 卡片       | 白底圆角卡片，轻阴影，信息密度适中             |
| 图标       | 线性图标（outline style），简洁克制            |
| 导航       | 顶部简洁导航栏 + 侧边（iPad）/ 底部（手机）Tab |
| 动效       | 几乎无动效，偶尔轻微的 fade/slide 过渡         |
| 设计关键词 | 高效、省时、一目了然                           |

#### 学生模式 — 科技 / 卡通 / 炫酷

| 要素       | 规范                                                    |
| ---------- | ------------------------------------------------------- |
| 整体风格   | 科技卡通风，类似《变形金刚：赛博坦》少年版              |
| 背景色     | 深蓝/深紫渐变（`#0F172A` → `#1E1B4B`），星空/电路板纹理 |
| 主色调     | 荧光蓝（`#00D4FF`）、能量橙（`#FF6B00`）                |
| 强调色     | 荧光绿（成功）、红色（警告），均为高饱和度              |
| 字体       | 带科技感的字体（如 Orbitron 或类似），中文用圆体/粗黑体 |
| 卡片       | 深色底 + 发光边框 + 微透明毛玻璃效果                    |
| 图标       | 填充式图标（filled），带发光/描边效果                   |
| 导航       | 底部 Tab 栏，带发光高亮效果                             |
| 动效       | 丰富的动效：粒子、发光、脉冲、悬浮                      |
| 音效       | 点击有反馈音效（嘀嗒/电子音）                           |
| 设计关键词 | 帅、酷、有趣、像在玩游戏                                |

#### 机甲设计

- **风格**：Low-Poly（低多边形）卡通风，辨识度高
- **实现**：SVG + CSS 渲染，无需外部美术资源
- **色彩**：金属蓝/银灰为底，能量核心用荧光色
- **细节**：关节处有发光线条，眼睛为两个发光点

### 12.2 关键动效清单

| 场景             | 动效描述                                   | 优先级 | 时长   |

## 13. 用户旅程

### 13.1 家长首次使用

```
1. 访问网站 → 点击"注册"
2. 输入邮箱 + 密码
3. 设置 4 位 PIN 码（提示："这个密码用来防止孩子进入管理界面"）
4. 输入孩子昵称（如"小明"）
5. 进入家长 Dashboard → 引导创建第一个任务（如"完成今日作业"，10 积分）
6. 引导创建第一个奖励（如"看一集动画片"，30 积分）
7. 提示"配置完成！点击右上角按钮切换到学生模式，把手机交给孩子吧！"
```

**目标：首次配置 < 5 分钟**

### 13.2 学生首次使用

```
1. 家长点击"切换到学生模式"
2. 过渡动画：白色界面淡出 → 深蓝色科技背景淡入
3. 完整引导动画（5s）：机甲从远处飞来 → 散落成零件
4. "欢迎，MotiMech · {昵称}！这是你的机甲，让我们唤醒它。"
5. 看到散落零件 + 机甲虚影轮廓
6. 下方显示任务列表 → "完成任务后让爸爸/妈妈来确认就能获得积分！"
7. 引导查看奖励商城 → "攒够积分就可以兑换奖励哦！"
```

**目标：首次体验 < 1 分钟，理解"做任务 → 得积分 → 机甲变化 + 换奖励"**

### 13.3 日常循环

**家长日常（每天 2~5 分钟）：**

```
打开 App（保持登录态，直接进入家长模式）
  → 查看 Dashboard（待确认事项有红色角标提示）
  → 确认孩子完成的任务（逐个或批量确认）
  → （如有）审核兑换请求
  → 点击"切换到学生模式" → 把设备递给孩子
```

**学生日常（每天 1~3 分钟）：**

```
从家长手中接过设备（已处于学生模式）
  → 看到机甲状态（有新积分时播放动画）
  → 查看今日任务列表和完成状态
  → （可选）逛奖励商城，规划兑换目标
  → 用完后还给家长 → 家长输入 PIN 切回
```

**也可能的场景（孩子主动拿设备）：**

```
孩子打开 App → 如果家长已登录且停留在家长模式
  → 看到 PIN 输入界面（无法进入家长功能）
  → 或者家长之前已切到学生模式 → 直接进入学生模式
```

### 13.4 关键里程碑

| 时间点    | 事件                       | 情感设计           |
| --------- | -------------------------- | ------------------ |
| Day 1     | 首次积分到账，脚部开始成型 | 好奇 + 期待        |
| Day 2~3   | 机甲"站起来"（阶段 2）     | 小小成就感         |
| Day 5     | 核心亮起（阶段 3）         | 越来越帅了！       |
| Day 14~18 | 机甲完整体亮相             | 强烈自豪感         |
| ~Week 2   | 首次兑换小奖励             | 积分"有用"的满足感 |
| ~Week 5   | 机甲首次进化               | 震撼 + 惊喜        |
| ~Month 3  | 兑换一个大奖励             | 长期坚持的回报     |

### 14.2 响应式规范

本产品为 **Web 应用**，需适配以下设备：

| 设备         | 屏幕宽度       | 布局策略                           |
| ------------ | -------------- | ---------------------------------- |
| 手机（竖屏） | < 640px        | 单列布局，底部 Tab 导航            |
| iPad / 平板  | 640px ~ 1024px | 双列卡片布局，侧边栏导航（可折叠） |
| 桌面（可选） | > 1024px       | 与 iPad 类似，内容区最大宽度限制   |

**通用设计要点：**

- **移动优先**：以手机和 iPad 为主要设计目标
- 触摸友好：按钮最小高度 44px，间距充足

**家长模式导航：**

- 手机：底部 Tab 栏（Dashboard / 任务 / 奖励 / 设置）+ 顶部右侧"切换"按钮
- iPad：左侧边栏导航 + 顶部右侧"切换"按钮
- 字体大小：正文 14~16px，紧凑但清晰

**学生模式导航：**

- 手机/iPad 统一使用底部 Tab 栏（机甲 / 任务 / 奖励 / 我的）
- 底部栏最右端有一个低调的小锁图标（切回家长模式）
- 机甲展示区域：在手机上占屏幕 50%~60% 高度
- 字体大小：正文不小于 16px，标题不小于 20px（更大更粗，适合儿童）

### 15.3 数据库 Schema

```prisma
model Parent {
  id            String    @id @default(cuid())
  email         String    @unique
  passwordHash  String
  pinHash       String                  // 4 位 PIN 码的哈希，用于学生模式切回
  createdAt     DateTime  @default(now())

  student       Student?
}

model Student {
  id            String    @id @default(cuid())
  nickname      String
  totalPoints   Int       @default(0)    // 累计获得积分（只增不减，驱动机甲进度）
  balance       Int       @default(0)    // 当前可用积分余额
  frozenPoints  Int       @default(0)    // 冻结积分（待确认兑换）
  streakDays    Int       @default(0)    // 连续活跃天数
  lastActiveAt  DateTime?
  createdAt     DateTime  @default(now())

  parentId      String    @unique
  parent        Parent    @relation(fields: [parentId], references: [id])

  taskLogs      TaskLog[]
  exchanges     Exchange[]
}

model Task {
  id            String    @id @default(cuid())
  name          String
  description   String?
  type          TaskType              // DAILY / WEEKLY
  points        Int                   // 完成奖励积分
  isActive      Boolean   @default(true)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  parentId      String
  taskLogs      TaskLog[]
}

enum TaskType {
  DAILY
  WEEKLY
}

model TaskLog {
  id            String    @id @default(cuid())
  completedAt   DateTime  @default(now())
  pointsAwarded Int

  taskId        String
  task          Task      @relation(fields: [taskId], references: [id])
  studentId     String
  student       Student   @relation(fields: [studentId], references: [id])

  @@unique([taskId, studentId, completedAt])
}

model Reward {
  id            String    @id @default(cuid())
  name          String
  description   String?
  points        Int                   // 兑换所需积分
  isActive      Boolean   @default(true)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  parentId      String
  exchanges     Exchange[]
}

enum ExchangeStatus {
  PENDING       // 待家长确认
  CONFIRMED     // 家长已确认
  REJECTED      // 家长已拒绝
}

model Exchange {
  id            String          @id @default(cuid())
  status        ExchangeStatus  @default(PENDING)
  pointsCost    Int                             // 兑换时的积分（快照）
  rejectReason  String?
  createdAt     DateTime        @default(now())
  confirmedAt   DateTime?

  rewardId      String
  reward        Reward          @relation(fields: [rewardId], references: [id])
  studentId     String
  student       Student         @relation(fields: [studentId], references: [id])
}

model PointsLog {
  id            String    @id @default(cuid())
  amount        Int                   // 正数=获得，负数=消耗
  type          PointsLogType
  description   String                // 来源描述（任务名/奖励名）
  createdAt     DateTime  @default(now())

  studentId     String
}

enum PointsLogType {
  TASK_REWARD       // 任务完成奖励
  EXCHANGE_COST     // 兑换消耗
  EXCHANGE_REFUND   // 兑换拒绝退回
}
```

### 15.4 核心 API 端点

| ---- | ---------------------- | ------------------------------------------------------------ |

| POST | `/api/mode/switch`     | 切换模式（parent → student 直接切；student → parent 需 PIN） |

| ------ | ----------------------------------- | -------------------- |

| PUT    | `/api/parent/pin`                   | 修改 PIN 码          |

| PUT    | `/api/parent/tasks/:id`             | 编辑任务             |
| DELETE | `/api/parent/tasks/:id`             | 删除任务             |
| POST   | `/api/parent/tasks/:id/confirm`     | 确认任务完成         |
| GET    | `/api/parent/rewards`               | 获取奖励列表         |
| POST   | `/api/parent/rewards`               | 创建奖励             |
| PUT    | `/api/parent/rewards/:id`           | 编辑奖励             |
| DELETE | `/api/parent/rewards/:id`           | 删除奖励             |

| POST   | `/api/parent/exchanges/:id/reject`  | 拒绝兑换             |
| GET    | `/api/parent/points-log`            | 积分流水             |

#### 学生模式 API

| 方法 | 端点                      | 说明                       |
| ---- | ------------------------- | -------------------------- |
| GET  | `/api/student/profile`    | 学生信息（积分、机甲状态） |

> 所有 API 都基于同一个登录 session 鉴权。模式状态（parent/student）存储在 session 或 cookie 中，家长模式 API 校验当前必须为家长模式，学生模式 API 校验当前必须为学生模式。

### 15.5 模式切换技术实现

| 方案              | 说明                                                                                                     |
| ----------------- | -------------------------------------------------------------------------------------------------------- |
| 模式状态存储      | Cookie（`mode=parent` 或 `mode=student`）                                                                |
| 切换到学生模式    | 前端调用 `/api/mode/switch`，服务端更新 cookie，前端跳转到 `/student`                                    |
| 切回家长模式      | 前端弹出 PIN 输入框 → 调用 `/api/mode/verify-pin` → 验证通过 → 更新 cookie → 跳转到 `/parent`            |
| 路由守卫          | Next.js middleware 检查 cookie 中的 mode，parent/_ 页面要求 mode=parent，student/_ 页面要求 mode=student |
| 直接访问 URL 保护 | 如果学生模式下直接输入 `/parent` URL → middleware 拦截 → 弹出 PIN 验证                                   |

```
用户浏览器（手机/iPad）
      │
      ▼

      │
      ▼
  Vercel Serverless Functions（Next.js API Routes）
      │
      ▼

```

- **零运维**：Vercel 自动处理构建、部署、HTTPS、CDN
- **自动扩容**：Serverless 函数按需执行
- **免费额度**：Vercel Hobby Plan 足够 MVP 阶段使用
- **数据库**：Vercel Postgres 基于 Neon，免费额度包含 256MB 存储

## 17. 风险与应对

| 风险                  | 影响                       | 应对措施                                            |
| --------------------- | -------------------------- | --------------------------------------------------- |
| **家长确认负担**      | 每天需多次确认，嫌麻烦     | 支持批量确认；Dashboard 一目了然待确认数量          |
| **学生无法自行操作**  | 积极性受限于家长响应速度   | 家长端做好通知提醒；UI 上让"待确认"状态对学生也可见 |
| **积分定价失衡**      | 任务奖励过高或奖品定价过低 | 提供定价建议参考；家长可随时调整                    |
| **机甲进度太快/太慢** | 失去目标感                 | 门槛值可在后续版本中支持家长微调                    |
| **孩子破解 PIN 码**   | 绕过家长控制               | 5 次错误锁定 1 分钟；PIN 修改入口在家长模式设置中   |
| **Vercel 免费额度**   | 用户增长后可能超限         | MVP 阶段足够；后续可升级 Pro Plan                   |
| **版权风险**          | "变形金刚"IP               | 全部原创机甲设计，不使用已有 IP                     |
