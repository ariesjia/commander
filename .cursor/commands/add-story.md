# Add Story

将用户想法细化为开发需求文档并落到 `docs/stories/`。

执行要求：

1. 先读取并遵循：`.cursor/skills/add-story/SKILL.md`
2. 按该 skill 的阶段流程执行（需求细化 → 代码库查询 → 技术方案 → 编号写入）
3. 生成完整 story 草稿后先给用户确认
4. 用户确认后创建文件：`docs/stories/000x-<name>.md`
5. 若存在相应章节，可同步更新 `docs/README.md` 的 story 索引

输入参数（若用户未提供需主动补齐）：

- 功能目标 / 用户价值
- 目标用户（家长端/学生端/管理员）
- 是否涉及数据库变更
- 期望优先级（可选）

输出格式要求：

- 严格使用 `add-story` skill 中的模板结构
- 必须包含：用户故事、验收标准、修改点、数据库设计、前端设计、风险与依赖
