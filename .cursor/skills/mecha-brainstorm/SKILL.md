# 机甲创意构思

为 MotiMech 系统构思全新机甲概念。先确认类型偏好，再基于现有设定设计与众不同的方案。

## 工作流程

### Phase 1: 确认机甲类型偏好

**必须先询问用户**，给出具体选项供选择：

```markdown
你想构思什么类型的机甲？请选择：

1. **人形机甲** - 类人形态，灵活作战（如玄甲、利刃、星盾）
2. **载具机甲** - 可乘载、形态偏载具（如空境、沧龙、镇岳）
3. **其他类型** - 机械兽、共生体、微型机甲、建筑型、后勤型等

也可以直接描述你心中的方向（如「想要一个地下的」「想要医疗辅助的」）。
```

**必须等用户明确选择或描述后**再进入 Phase 2。

### Phase 2: 读取现有设定

读取 `docs/MECHA_CHARACTERS.md`，梳理当前机甲类型与定位：

| 类型 | 已有机甲 | 定位/职责 |
|------|----------|-----------|
| 人形 | 星盾、玄甲、利刃、风驰、雷啸、磁暴 | 防御、强攻、突袭、侦查、爆发、磁场控制 |
| 载具 | 空境、沧龙、镇岳 | 空中、深海两栖、陆地堡垒 |

**设计原则**：新机甲必须与以上**完全不同**——避开已有定位，选择小众、独特的类型。

### Phase 3: 构思 2～3 个方案

为每种类型提供**至少 2 个、至多 3 个**完整概念，格式如下：

```markdown
## 方案 A：[中文名] (English)

- **类型**：[具体定位，如「地下隧道载具机甲」]
- **特点**：[1-2 句话，核心能力与差异化]
- **职责**：[职责列表，逗号分隔]
- **配色**：[主色 + 辅色]
- **专属装备**：[1-2 个核心装备名]
- **简介**：[1-2 句话介绍，可用于 intro]
- **等级称号**（0-7）：量产型、改良型、特装型、强化型、[自定义4]、[自定义5]、[自定义6]、完整体

---
```

**差异化要求**：
- 人形：避开 防御/强攻/突袭/侦查/爆发/磁场，可考虑：医疗、后勤、信息、隐形、增益、召唤等
- 载具：避开 空中/深海/陆地堡垒，可考虑：地下、沙漠、极地、医疗运输、工程建造等
- 其他：机械兽、共生体、微型、建筑型、时间/空间系等小众方向

### Phase 4: 展示并等待选择

将 2-3 个方案完整展示给用户，询问：

```markdown
以上方案中，你更倾向哪一个？可以选一个深入细化，或提出修改意见。
```

## 小众类型参考（避免与现有重复）

- 地下/隧道型、沙漠/极地型
- 医疗/后勤/补给型
- 隐形/伪装/侦察型
- 机械兽/仿生型
- 建筑/工程/建造型
- 信息/电子战/黑客型
- 增益/辅助/光环型
- 微型/单兵/渗透型
